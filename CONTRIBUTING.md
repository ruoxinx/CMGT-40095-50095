# Contributing to AI in Construction

Thank you for your interest in contributing to these course materials.

## Ways to contribute

- improve teaching materials or module pages
- suggest clearer explanations for students with limited coding background
- contribute datasets, case studies, or examples relevant to AEC
- share guest lecture ideas, slides, demos, or industry perspectives
- report broken links, setup friction, or accessibility issues

## Recommended contribution workflow

1. Fork the repository.
2. Create a focused branch for your change.
3. Make updates that are clear, student-friendly, and well organized.
4. If relevant, add or update documentation in `docs/`.
5. Open a pull request with a short explanation of what changed and why it helps learners.

## Content guidelines

- prefer plain language over jargon when possible
- connect technical methods to construction or built-environment practice
- keep student usability in mind, especially for first-time notebook users
- use accessible headings, descriptive links, and readable structure
- avoid adding copyrighted third-party materials unless you have permission

## Good contributions for this repository

- new module pages or improved module descriptions
- improved Google Colab launch workflows
- assignment clarifications or starter templates
- project examples and case studies
- guest speaker summaries or public resource links
- corrections to course navigation or public website content
